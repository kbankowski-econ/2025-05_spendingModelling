## ✅ Done since 2026-06-28

**§4.3 Sensitivity analysis — the main thread**

- **Why not Dynare's native GSA.** The `sensitivity`/`dynare_sensitivity` toolbox
  is built on the perturbation (`stoch_simul`) solution; this model is solved by
  `perfect_foresight_solver` throughout, so the toolbox does not apply without a
  separate stationarized model, and it answers a different question (determinacy +
  moment Sobol) than §4.3 needs. The right tool is a **bespoke one-at-a-time (OAT)
  parameter sweep** — the same pattern as `investigations/galiCheck/phi_sweep.m`.
- **New investigation:** `docs/2026-06_wp-imf/investigations/sensitivity/`
  (`README.md` explains the rationale). `sweep.m` builds each AE expansion in a
  gitignored `work/` dir (so the committed `models/**/_results.mat` are never
  touched — shared macros pulled via the preprocessor `-I` path), then sweeps
  parameters with `set_param_value` + re-solve, recomputing the **Table-3
  present-value own-spending multiplier** and the **annual `yd` IRF** per draw.
  Outputs: `results/sweep_AE.csv` (multipliers) and `results/sweep_AE_irf.csv`
  (annual IRFs); `plot_sweep.py` makes diagnostic grid/tornado/IRF-fan PNGs.
- **Parameters swept (AE):** α_G (`alphaG`), μ (`alphaH`), α_RD (`alphaRD`,
  derived = 0.09·(1−ρ_A) = 0.0189, read from `M_.params` at build time), and the
  adoption elasticity ς (`rhoSADOPT`). **Efficiency wedges e^GI/e^GE/e^GRD were
  dropped** — the per-dollar multiplier is *exactly* invariant to them (the (1−e)
  wedge scales effective spending and the steady-state stock by the same factor,
  cancelling in the ratio). That is the clean §4.3 ↔ §5.2 separation: gaps set the
  *stock*, not the *marginal* multiplier.
- **Findings (validated — baselines reproduce Table 3 exactly).** Each multiplier
  is driven mainly by its own elasticity: long-run output multiplier spans ≈0–12
  (infra, in α_G), ≈3–26 (HC, in μ), ≈1–34 (R&D, in α_RD). **ς is the one systemic
  parameter** — it moves every multiplier at long horizons, strongly convex above
  ς≈0.8, and *reshapes the R&D path* (low ς front-loads, high ς back-loads). The
  paper calibrates ς=0.8 while the literature suggests 0.9–0.95, so it sits at the
  knee. Government consumption (≈−1.2) responds only to ς.
- **Paper figure (Figure 2, §4.3 `fig:sensitivity`).** New
  `pyScripts/2026-06_wp-imf/plotSensitivityIRF.py`, built with the **standard chart
  pipeline** (plotly + `wp_charts` helpers + `chartTable.csv` sizing + `tasks.py`
  task in `run_all`), reading the committed `sweep_AE_irf.csv`. A 1×3 output-IRF
  fan (Infrastructure/α_G, Human Capital/μ, R&D/α_RD): light→dark lines over each
  parameter's range, **thick grey baseline** (no legend — identified in the note),
  per-line value labels at the right, **shared y-axis** across panels, and a
  vertical de-collision that spreads the compressed infrastructure labels outward
  from the (fixed) baseline label. §4.3 prose rewritten around the figure; the
  efficiency wedges reframed as scaling the stock (link to §5.2).
