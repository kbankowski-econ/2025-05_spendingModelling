### Working in parallel (git worktrees)
When running several agents at once, give each its own worktree so they never
clobber each other's `draftPaper.tex`, figures, or `_results.mat`:
- Create: `bashScripts/wt-new.sh <name>` → a sibling checkout at
  `../sm-worktrees/<name>` on a fresh branch off `main`. The script rewrites the
  gitignored `+utils/+call/paths.m` so MATLAB/Dynare write into the worktree, not
  the main repo (see project `CLAUDE.md` → *Git Worktrees*).
- One agent per worktree/branch; split the TODO list so tasks don't overlap (e.g.
  one agent on equations, one on figures). Each worktree shares the LFS store but
  has its own working files (~1.5 GB smudged), so remove them when done:
  `bashScripts/wt-rm.sh <name>`.
- Merge each branch back to `main` separately; resolve `draftPaper.tex` conflicts
  by section. Avoid two agents editing the same equation block or figure pipeline
  concurrently — coordinate via the TODO list above before fanning out.

### Step by step (example: `wp-equations` + `wp-literature`)

**1. Create one worktree per task** (from the main checkout):
```bash
# cwd = /Users/kk/Developer/2025-05_spendingModelling
bashScripts/wt-new.sh wp-equations    # branch wp-equations, ../sm-worktrees/wp-equations
bashScripts/wt-new.sh wp-literature   # branch wp-literature, ../sm-worktrees/wp-literature
git worktree list                     # verify
```

**2. Start one tmux session per worktree** — each runs an agent, detached:
```bash
tmux new-session -d -s wp-equations  -c /Users/kk/Developer/sm-worktrees/wp-equations  claude
tmux new-session -d -s wp-literature -c /Users/kk/Developer/sm-worktrees/wp-literature claude
tmux ls                              # list the running sessions
tmux attach -t wp-equations          # attach; detach with C-b d, switch with C-b s
tmux attach -t wp-literature          # attach; detach with C-b d, switch with C-b s
```
`-s` names the session, `-c` sets its working dir, and the trailing `claude`
is the command it runs. Give each the matching TODO: wp-equations → equations
consistent with the paper (notation, steady state, efficiency gaps);
wp-literature → model consistent with the literature. Kill a finished session
with `tmux kill-session -t wp-equations`.

**3. Let them work.** Each agent commits to its own branch inside its own folder
(`git commit` as normal) — files are physically separate, so no clobbering.

**4. Merge back, one branch at a time** (from the main checkout); merge the
smaller branch first so the second merge's conflicts are smaller:
```bash
cd /Users/kk/Developer/2025-05_spendingModelling
git merge wp-equations
git merge wp-literature                # resolve draftPaper.tex conflicts by section
```

**5. Tear down:**
```bash
bashScripts/wt-rm.sh wp-equations      # removes folder + merged branch
bashScripts/wt-rm.sh wp-literature     # add --keep-branch to keep a branch
```

Variations: fork from a non-`main` base with
`bashScripts/wt-new.sh wp-hotfix some-feature-branch`. Parallel MATLAB runs are
safe — each worktree's `paths.m` points at itself, so `_results.mat` land in
separate trees.

  - ✅ **Gov-consumption deflation puzzle — RESOLVED (2026-06-25).** The deflation is the
    standard New Keynesian *wealth effect on labor supply* (Galí–López-Salido–Vallés 2007's
    motivating puzzle), not a bug or a non-standard-feature artifact. A permanent, wasteful,
    lump-sum-financed `Gc` increase lowers lifetime resources → households work more, the
    real wage = marginal cost falls → the NKPC delivers deflation. Demonstrated by three
    appendix figures (App. C, all on the from-scratch canonical NK `Model_NK_exp_gc*`, built
    from `modelTemplateNK.mod`; plotted by `pyScripts/2026-06_wp-imf/plot{Simplified,Duration,

    · `fig:simplifiedGc` (Fig 8) — full model → 4 simplifications → canonical NK; deflation
      survives every step, so it is a textbook-NK property under a permanent shock.
    · `fig:durationGc` (Fig 9) — on/off shocks of 1q/1y/5y/permanent (`_d1`/`_d4`/`_d20` +
      permanent). Short shocks are *inflationary* on impact (small wealth effect → demand
      dominates); the permanent one gives the full −0.92 ppt deflation.

    over 60 quarters** (≈ machine/solver precision) for every variable — our NK is

    Calvo form and the `omega` labor-weight normalization are algebraically equivalent). The
    Galí baseline DRS case (α=1/4) differs only in the *magnitude* of the real-wage /
    marginal-cost / hours response (steeper, decreasing returns) — **same signs**.
  - ⚠ **Key finding — the deflation's sign is calibration-dependent (Frisch × persistence).**
    The cross-check exposed that under **Galí's own calibration** (φ=5, i.e. Frisch 0.2) a
    *persistent* AR(1) `ρ=0.9` `Gc` shock is **inflationary** (+0.11 impact), not deflationary.
    Sweeping φ in the verified model (all else at Galí) gives impact inflation (percent dev):

    | φ (Frisch) | ρ=0.9 | ρ=0.99 |
    |---|---|---|
    | **5.0 (0.20) — Galí** | **+0.114** | −0.029 |
    | 2.0 (0.50) | +0.041 | −0.076 |
    | **1.2 (0.83) — ours** | −0.003 | −0.108 |
    | 0.5 (2.0) | −0.068 | −0.159 |
    | 0.2 (5.0) | −0.110 | −0.195 |

    So the sign is governed **jointly** by (i) persistence — more persistent ⇒ more
    deflationary (matches Figs 9–10) — and (ii) the Frisch elasticity — more elastic labor
    (lower φ) ⇒ more deflationary. The deflation needs the wealth effect to actually shift
    labor supply, which requires both elastic labor *and* a large PV of the spending. At
    Galí's textbook Frisch (0.2) a typical transitory stimulus is inflationary; deflation
    appears only near-permanent. The paper's "even canonical NK deflates" claim is correct
    **for the permanent shock** (used in `fig:standardShocks` + the simplification figure)
    but is **not universal**. Our calibration (φ=1.2, plus `γy=0.25`/`θ=0.8`) sits on the
    deflationary side; pure Galí does not at ρ=0.9.

## Writing style — drafting the paper (esp. the introduction)
Follow Claudia Sahm, *Writing Economics Research Papers* (2021), in
`literature/ClaudiaSahm_WritingEcon.pdf` — a style reference for drafting, **not a
citation** (do not add it to `references.bib` or cite it in the text).

**How the introduction should be written.** Each element below is **one paragraph**
in the introduction (and **one sentence each** in the abstract), roughly in order:
1. **Motivation** — why the question matters. Set the stage with *facts*; do not just
   assert that it is important.
2. **Research question** — state one *specific* question ("This paper studies / asks
   whether …").
3. **Contribution** — what answering the question teaches us. **Lead with our own
   contribution**, then place it in the context of others. The contribution paragraph
   *substitutes for* a related-literature section; if one is unavoidable, keep it short.
4. **Methods** — how the question is answered: identification and the key features of
   the approach.
5. **Results** — the main findings, **no more than three**, stated clearly and with
   *numbers*. These are the payoff — take the space to explain them.
6. **Road map** — a short by-section description of the rest of the paper.

**Make contribution and findings shine — they are the most important parts.**

General style rules to apply throughout:
- First sentence of each paragraph makes its main point; read in sequence, the opening
  sentences should tell the paper's story.
- Stick to facts; avoid subjective words ("surprising", "important", "obvious") and
  limit adverbs.
- Emphasize *economic* significance over statistical significance.
- No jargon without first defining it (readers come from other sub-fields).
- Figures/tables must stand alone (no acronyms/jargon/symbols), with labelled axes,
  legible fonts, and legibility in black and white; keep a "money chart" for the
  headline result.
- Do not apologize for what the paper does not do.

## IMF Editorial Style Guide (house style)
Reference: `literature/IMF_EditorialStyleGuide_2e.pdf` (2024, 2nd ed.) — IMF house
style for the WP. **Not a citation** (do not add to `references.bib` or cite). Rules
that bear on this paper:
- **Numbers:** spell out < 10, numerals for ≥ 10 (mixed → numerals); always numerals
  for percentages, ratios, decades, currency-with-symbol, and millions and above.
  Use the word "percent" in text, not "%".
- **Ranges & punctuation:** en dash for ranges (2026–31; 6–8 percent). Use the serial
  (Oxford) comma. Hyphenate phrasal adjectives before a noun (long-run output,
  debt-to-GDP ratio, spending-efficiency gap).
- **Cross-references:** capitalize in-text refs to Figure / Table / Section /
  Appendix / Box, but **not** numbered equations or figure panels — "equation (2.10)",
  "panel a" (lowercase). Use `\eqref{}` for equations.
- **Titles:** headline-style (Title Case) for section, figure, and table titles
  (matches CLAUDE.md "Title Case for chart titles").
- **Country terms (lowercase in running text):** advanced economies; emerging market
  and developing economies (EMDEs); low-income (developing) countries. Avoid "advanced
  countries" and "emerging economies". GDP and R&D always as acronyms.
- **Currency:** "$" is the assumed US dollar; use "US$" only to disambiguate. "US",
  never "U.S.".
- **Figures/tables:** Source/Sources line first, then a "Note:" line; titles
  headline-style; must stand alone (no undefined symbols/jargon).
