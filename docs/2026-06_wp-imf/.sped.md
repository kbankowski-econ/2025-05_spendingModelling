### Working in parallel (git worktrees)
When running several agents at once, give each its own worktree so they never
clobber each other's `draftPaper.tex`, figures, or `_results.mat`:
- Create: `bashScripts/wt-new.sh <name>` → a sibling checkout at
  `../sm-worktrees/<name>` on a fresh branch off `main`. The script rewrites the
  gitignored `+utils/+call/paths.m` so MATLAB/Dynare write into the worktree, not
  the main repo (see project `CLAUDE.md` → *Git Worktrees*).
- One agent per worktree/branch; split the TODO list so tasks don't overlap (e.g.
  one agent on equations, one on figures). Each worktree shares the LFS store but
  has its own working files (~1.5 GB smudged), so remove them when done:
  `bashScripts/wt-rm.sh <name>`.
- Merge each branch back to `main` separately; resolve `draftPaper.tex` conflicts
  by section. Avoid two agents editing the same equation block or figure pipeline
  concurrently — coordinate via the TODO list above before fanning out.

### Step by step (example: `wp-equations` + `wp-literature`)

**1. Create one worktree per task** (from the main checkout):
```bash
# cwd = /Users/kk/Developer/2025-05_spendingModelling
bashScripts/wt-new.sh wp-equations    # branch wp-equations, ../sm-worktrees/wp-equations
bashScripts/wt-new.sh wp-literature   # branch wp-literature, ../sm-worktrees/wp-literature
git worktree list                     # verify
```

**2. Open one agent per worktree** — a separate terminal for each:
```bash
# terminal A
cd /Users/kk/Developer/sm-worktrees/wp-equations && claude
# terminal B
cd /Users/kk/Developer/sm-worktrees/wp-literature && claude
```
Give each the matching TODO: A → equations consistent with the paper (notation,
steady state, efficiency gaps); B → model consistent with the literature.

**3. Let them work.** Each agent commits to its own branch inside its own folder
(`git commit` as normal) — files are physically separate, so no clobbering.

**4. Merge back, one branch at a time** (from the main checkout); merge the
smaller branch first so the second merge's conflicts are smaller:
```bash
cd /Users/kk/Developer/2025-05_spendingModelling
git merge wp-equations
git merge wp-literature                # resolve draftPaper.tex conflicts by section
```

**5. Tear down:**
```bash
bashScripts/wt-rm.sh wp-equations      # removes folder + merged branch
bashScripts/wt-rm.sh wp-literature     # add --keep-branch to keep a branch
```

Variations: fork from a non-`main` base with
`bashScripts/wt-new.sh wp-hotfix some-feature-branch`. Parallel MATLAB runs are
safe — each worktree's `paths.m` points at itself, so `_results.mat` land in
separate trees.
