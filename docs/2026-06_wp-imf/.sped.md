### Working in parallel (git worktrees)
When running several agents at once, give each its own worktree so they never
clobber each other's `draftPaper.tex`, figures, or `_results.mat`:
- Create: `bashScripts/wt-new.sh <name>` ŌåÆ a sibling checkout at
  `../sm-worktrees/<name>` on a fresh branch off `main`. The script rewrites the
  gitignored `+utils/+call/paths.m` so MATLAB/Dynare write into the worktree, not
  the main repo (see project `CLAUDE.md` ŌåÆ *Git Worktrees*).
- One agent per worktree/branch; split the TODO list so tasks don't overlap (e.g.
  one agent on equations, one on figures). Each worktree shares the LFS store but
  has its own working files (~1.5 GB smudged), so remove them when done:
  `bashScripts/wt-rm.sh <name>`.
- Merge each branch back to `main` separately; resolve `draftPaper.tex` conflicts
  by section. Avoid two agents editing the same equation block or figure pipeline
  concurrently ŌĆö coordinate via the TODO list above before fanning out.

Examples:
```bash
# Two agents, two tasks ŌĆö one on equations, one on figures
bashScripts/wt-new.sh wp-equations     # ŌåÆ ../sm-worktrees/wp-equations (branch wp-equations)
bashScripts/wt-new.sh wp-figures       # ŌåÆ ../sm-worktrees/wp-figures  (branch wp-figures)

# Point each agent/terminal at its own checkout
#   agent A: cwd = ../sm-worktrees/wp-equations  ŌåÆ edits eq. (Z), ┬¦2.2
#   agent B: cwd = ../sm-worktrees/wp-figures    ŌåÆ regenerates efficiency*.png

# When done, merge each branch back and tear the worktrees down
git -C . merge wp-equations            # run from the main checkout
git -C . merge wp-figures
bashScripts/wt-rm.sh wp-equations      # removes worktree + branch
bashScripts/wt-rm.sh wp-figures

# Fork from a non-main base, or keep the branch after removing the worktree
bashScripts/wt-new.sh wp-hotfix some-feature-branch
bashScripts/wt-rm.sh wp-hotfix --keep-branch
```
